Using the SGUI library

Filesystem abstraction

It has already been mentioned that SGUI supports filesystem abstraction (the NULL pointer passed to the sgui_font_load function). What this means is that SGUI does not access the filesystem directly, instead it uses a struct of type sgui_filesystem, holding various callbacks.

With this meachanism you can redirect the SGUI file accesses to your own functions, so it accesses a compressed archive, a network source, a virtual filesystem like PhysicsFS or some implementation for a microcontroller where there is no native filesystem instead of the actual filesystem.

All you need to do is instantiate the sgui_filesystem structure, fill it with pointers to functions that open/close/read from/get the size of opaque file handles. (Consult the doxygen documentation or read the sgui_filesystem.h header for more details).

All SGUI functions that access files require a pointer to an sgui_filesystem structure. If you pass NULL, they simply call sgui_filesystem_get_default what returns a pointer to an implementation that uses the STDIO functions.
