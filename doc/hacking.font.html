Font rasterization

The backend also has to supply an implementation of the sgui_font data structure. The functions from sgui_font.h are implemented directly in the backend.

The implementations in the Windows® and X11 backends use the FreeType library to rasterize true type fonts, but a backend might also implement a simple, static font bitmap.

The sgui_font functions support the following operations:
  • Load fonts from a file or from memory.
  • Get the dimensions of a glyph.
  • Render a glyph to a memory buffer.
Those functions are then used by canvas implementations to rasterize text. However, a canvas might also use an sgui_font_cache that caches glyph in an sgui_pixmap to minimize glyph loading and rasterizing and instead blend the already loaded glyphs. The font cache uses an RB-Tree internally to store loaded glyphs and theire area on the cache pixmap. It only loads glyphs if they cannot be found in the tree.
